# 分析Log v1.0

整骨院・整体院・理学療法士・トレーナー向けの姿勢分析／動作分析アプリ。
スマホ・iPadのブラウザで動きます（インストール不要・ビルド不要）。

公開URL: https://tokitoa-coder.github.io/bunseki-log/

## 機能

### ① 撮影
- 全画面カメラ（前後切替）
- 中心線・グリッド（3×3／5×5／8×8／16×16）
- 水平器（端末の傾きに連動、水平で緑）
- 端末内の動画を読み込むことも可能

### ② 再生・切り出し
- 全画面プレイヤー、スロー再生（0.25〜1.0倍）、コマ送り／コマ戻し
- 静止画の切り出し（自動保存・撮影日時つき）

### ③ 比較
- Before / After（横並び・重ね（透過）・スライダー）
- 患者ごとの履歴（日付順、その場でBefore/Afterに設定）
- レポート作成（印刷・PDF保存）
- バックアップ（書き出し／読み込み）

### 画像編集（画像をタップ）
- 中心線のドラッグ、グリッド切替、移動・ピンチズーム
- 描画（自由線・直線・水平・垂直・矢印・円・文字／色・太さ）
- 測定（左右差・距離・角度・cm基準の登録）
- AI姿勢解析（MediaPipe Pose）
  - 肩の高さ、骨盤の傾き、体幹、頭部位置、膝角度、左右差
  - ランドマークを手でドラッグ修正でき、数値はその場で再計算
  - cm基準を登録すると cm／mm 表示（未登録時は角度のみ）

## 運用上の注意

- 画像は端末のブラウザ内に保存されます。クラウド同期はありません。
- **月1回は「ファイルに書き出す」でバックアップ**を取ってください（iCloud Drive推奨）。
- 7日以上アプリを開かないと、ブラウザがデータを削除することがあります。
- 大切なBefore/Afterは「保存」で写真アプリにも残してください。
- iPadとiPhoneのデータは別々です。移すときはバックアップの書き出し／読み込みを使用。
- 患者情報は実名ではなく管理番号での運用を推奨します。
- AIの計測値は推定です。骨格が体に合っているか確認したうえで、説明の目安としてお使いください。

## 更新方法

`index.html` 1ファイルのみで動作します。差し替えるときは、
GitHubの `index.html` を新しいものに置き換えてください（URL・QRコードは変わりません）。

※ iPadでダウンロードすると `index 2.html` のような別名になることがあります。
アップロード前にファイル名が `index.html` であることを確認してください。

## 技術構成

- HTML / CSS / JavaScript（フレームワーク・ビルド不要の単一ファイル）
- 保存：IndexedDB（画像）／localStorage（設定）
- AI：MediaPipe Pose（CDNから読み込み・解析時のみ通信が必要）
- 公開：GitHub Pages
